RIDDLE E1: The Voting Logicians (Common Knowledge + Game Theory)
------------------------------------------------------------------------
SETUP:
Three logicians (A, B, C) privately vote Yes or No on a proposal. Majority wins.
After voting, each is told: "At least one person voted the same as you."
All three logicians then say: "I now know the exact vote count."

THE QUESTION: What was the vote?

HINT: What does unanimous knowledge tell you?

ARCHETYPE FUSION: Common Knowledge (what everyone knows everyone knows) +
Game Theory (each person reasons about others' information)

SOLUTION:
Step 1: Possible outcomes: 3-0 (all Yes), 2-1, 1-2, 0-3 (all No).
Step 2: Each person hears "at least one voted same as you."
        - If you voted Yes and hear this: at least one other Yes.
        - If you voted No and hear this: at least one other No.
Step 3: This rules out 3-0 and 0-3? No! If 3-0, everyone voted Yes, everyone hears
        "at least one same" (true, all three same). Same for 0-3.

Step 4: After hearing this, when can someone KNOW the exact count?
        - If 3-0: You voted Yes, know ≥1 other Yes. Could be 2-1 or 3-0. NOT certain.
        - If 2-1: The Yes voters know ≥1 other Yes (could be 3-0 or 2-1). NOT certain.
                  The No voter knows ≥1 other No... but there ISN'T another No! CONTRADICTION.

Step 5: Wait - if the vote is 2-1, the sole No voter is told "at least one voted same."
        But no one else voted No. This statement would be FALSE.
        So the announcement would be different for that person.

REREAD: "Each is told: 'At least one person voted the same as you.'"
If 2-1: Two people voted Yes (true for them), one voted No (FALSE for them).
The No person would be told something false? Or the puzzle assumes announcements are true?

INTERPRETATION: The announcement is always TRUE. So 2-1 and 1-2 are impossible.
(The minority voter has no one voting same as them.)

Step 6: Only 3-0 or 0-3 remain.
Step 7: If 3-0: Each voted Yes, hears "≥1 same." Each knows outcome could be 3-0.
        But could it be 2-1? No, because in 2-1, minority gets false announcement.
        So each person, hearing the (true) announcement, deduces NOT 2-1.
        Everyone voted Yes → 3-0. CERTAIN.
Step 8: Same logic for 0-3.

But wait - how do they know if it's 3-0 or 0-3? They know their own vote!
- If you voted Yes and hear true announcement: must be 3-0.
- If you voted No and hear true announcement: must be 0-3.

ANSWER: The vote was unanimous (either 3-0 or 0-3).

But the puzzle asks "what was the vote" not "what structure." Without knowing
a specific person's vote, we can only say it was unanimous.

VERIFICATION:
- 3-0: All told "≥1 same" (true). Each deduces 3-0. ✓
- 0-3: All told "≥1 same" (true). Each deduces 0-3. ✓
- 2-1: One person told false statement. Violates puzzle setup.
- 1-2: Same problem.

ANSWER: Vote was unanimous. All three know this.

ARCHETYPE SCORES:
- Originality: 8/10 (novel fusion)
- Cleverness: 8/10 (the elimination of 2-1 is elegant)
- Simplicity: 7/10 (requires careful reading)
- Solution Success: 8/10 (complete but answer is "unanimous" not specific)

TOTAL: 31/40

RIDDLE E2: The Predicting Prisoners (Self-Reference + Probability)
------------------------------------------------------------------------
SETUP:
100 prisoners each secretly write a prediction for how many total prisoners will
predict "greater than 50." Then everyone's prediction is revealed simultaneously.
Any prisoner whose prediction matches the actual count goes free.

THE QUESTION: What should you predict?

HINT: Your prediction affects the count your prediction is about.

ARCHETYPE FUSION: Self-Reference (prediction affects outcome being predicted) +
Probability (what's the equilibrium?)

SOLUTION:
Step 1: Let X = number of prisoners who predict "greater than 50."
Step 2: If you predict "greater than 50," you contribute to X.
Step 3: If X > 50, then predicting "greater than 50" would be correct.
        If X ≤ 50, then predicting "50 or less" (i.e., anything ≤ 50) would be correct.

Step 4: Consider symmetric equilibrium where each prisoner predicts "51" with
        probability p. No wait, they predict whether count exceeds 50, not exact count.

REREAD: They predict the count, not a binary. "How many total will predict >50."
So predictions are numbers 0-100.

Step 5: If everyone predicts "51": X = 100 (everyone predicted >50). Actual = 100. No match.
        If everyone predicts "0": X = 0. Actual = 0. MATCH! Everyone goes free.

Step 6: But wait - predicting "0" means predicting zero people will predict >50.
        If everyone predicts 0, how many predicted >50? Zero.
        Prediction of 0 matches actual of 0. ✓

ANSWER: Everyone should predict "0". Then X = 0, and all predictions match.

But is this stable? If 99 people predict 0, what should I do?
- If I predict 0: X = 0, my prediction matches. ✓
- If I predict 1: X = 1, my prediction matches. ✓
- If I predict 50: X = 1 ≠ 50. ✗

Actually both 0 and 1 work for me. But which is "better"?
If we're maximizing my chance of freedom:
- Predicting X-1 where X is actual count also works if I change the count by leaving.

Hmm, this is getting complex. Let's simplify:

FOCAL POINT SOLUTION: "Predict 0" is a focal equilibrium.
- If everyone predicts 0: X = 0, all predictions correct.
- Any deviation (predicting >50) makes X > 0, deviator is wrong if others stay at 0.

ANSWER: Predict 0.

VERIFICATION:
- All predict 0 → X = 0 → all correct. ✓
- If one deviates to >50: X = 1 → deviator wrong (predicted >50, actual = 1). ✓
- Equilibrium is stable.

ARCHETYPE SCORES:
- Originality: 9/10 (novel self-referential setup)
- Cleverness: 8/10 (the focal point insight is good)
- Simplicity: 7/10 (rules need careful parsing)
- Solution Success: 9/10 (verified equilibrium)

TOTAL: 33/40 [BEST SO FAR!]

RIDDLE E3: The Shrinking Room (State Space + Bayesian)
------------------------------------------------------------------------
SETUP:
You're in a room that shrinks by 10% each hour. You can escape by finding a hidden
key. Each hour, you search for the key with 20% success rate if it's in your half
of the room, 0% if it's in the other half.

The key was placed randomly in the room (50/50 either half). If you successfully
search your half and find nothing, the key must be in the other half.

THE QUESTION: In which half should you search each hour to maximize survival?

HINT: Update your beliefs about where the key is.

ARCHETYPE FUSION: State Space (room shrinking = time pressure) +
Bayesian (updating key location belief)

SOLUTION:
Step 1: Initial belief: P(key in half A) = P(key in half B) = 50%.
Step 2: Hour 1: Search half A.
        - If find key: escape! (20% of 50% = 10% chance)
        - If don't find key: Either key wasn't there (50%) OR was there but missed (80% of 50% = 40%)
Step 3: After failed search of A:
        - P(key in A | searched A, nothing) = P(missed | in A) × P(in A) / P(nothing)
        - P(nothing) = P(not in A) + P(in A) × P(miss) = 0.5 + 0.5 × 0.8 = 0.9
        - P(key in A | nothing) = (0.8 × 0.5) / 0.9 = 0.4 / 0.9 ≈ 0.444
        - P(key in B | nothing) ≈ 0.556

Step 4: Hour 2: Should search B (more likely).
        After failed search of B:
        - P(key in B | searched B, nothing) = (0.8 × 0.556) / P(nothing)
        - P(nothing) = P(not in B) + P(in B) × 0.8 = 0.444 + 0.556 × 0.8 = 0.889
        - P(key in B | nothing) = (0.8 × 0.556) / 0.889 ≈ 0.500
        - P(key in A | nothing) ≈ 0.500

Step 5: After searching each half once, we're back to 50/50!
        The optimal strategy is to alternate, always searching the more likely half.

ACTUALLY, let's reconsider. With shrinking room:
- Maybe after some hours, the room is too small regardless?
- The shrinking adds urgency but doesn't change Bayesian calculations.

SIMPLIFIED ANSWER: Alternate halves, starting with either.
After searching A and failing: B is more likely. Search B.
After searching B and failing: they're equal. Could go either way.

FULL OPTIMAL: Search whichever half currently has higher probability.
Since failed searches update probabilities toward the unsearched half,
this means alternating is optimal.

ANSWER: Alternate between halves each hour.

VERIFICATION:
- Hour 1: Search A (50%). Find or update.
- Hour 2: If failed, B is 55.6%. Search B.
- Hour 3: If failed, back to ~50/50. Either works.
- Alternating gives best chance of finding before room kills you.

ARCHETYPE SCORES:
- Originality: 7/10 (Bayesian search is known, shrinking room adds urgency)
- Cleverness: 7/10 (the probability update is nice)
- Simplicity: 6/10 (setup is somewhat complex)
- Solution Success: 8/10 (correct but not deeply surprising)

TOTAL: 28/40
